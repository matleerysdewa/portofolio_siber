SQL Injection adalah salah satu kerentanan web paling umum dan berbahaya. Di artikel ini saya bahas dari dasar: apa itu SQLi, kenapa bisa terjadi, dan bagaimana cara mengeksploitasinya secara etis menggunakan lab sendiri...

Apa itu SQL Injection?

SQL Injection (SQLi) adalah teknik serangan yang memungkinkan attacker menyisipkan atau "meng-inject" kode SQL berbahaya ke dalam query database melalui input yang tidak tervalidasi. Akibatnya, attacker bisa membaca data sensitif, mengubah data, bahkan mengambil alih seluruh server.

"SQL Injection masih menduduki posisi teratas OWASP Top 10 — artinya ini masih ancaman nyata sampai sekarang."

Kenapa SQLi Bisa Terjadi?

Penyebab utamanya adalah developer yang langsung menggabungkan input user ke dalam query SQL tanpa sanitasi. Contoh kode PHP yang rentan:

// RENTAN — jangan pernah lakukan ini!
$username = $_POST['username'];
$query = "SELECT * FROM users WHERE username = '$username'";

Kalau user mengisi ' OR '1'='1, query yang dieksekusi jadi:

SELECT * FROM users WHERE username = '' OR '1'='1'

Karena '1'='1' selalu TRUE, query ini mengembalikan semua user di database!

Cara Eksploitasi (Lab Environment)

Saya menggunakan DVWA (Damn Vulnerable Web Application) sebagai target latihan. Berikut langkah-langkahnya:

# 1. Cek apakah form rentan
' -- -

# 2. Cari jumlah kolom
' ORDER BY 1-- -
' ORDER BY 2-- -
' ORDER BY 3-- -   <-- error di sini = 2 kolom

# 3. Dump data dengan UNION
' UNION SELECT username, password FROM users-- -

Gunakan sqlmap untuk Automasi

sqlmap -u "http://target.com/page?id=1" --dbs --batch
⚡ Ingat: Selalu lakukan ini di lab sendiri atau dengan izin tertulis. SQL Injection tanpa izin adalah kejahatan.

Cara Mencegah SQLi

  • Gunakan Prepared Statements / Parameterized Queries
  • Gunakan ORM (Eloquent, Hibernate, dll)
  • Validasi dan sanitasi input user
  • Batasi privilege user database
  • Terapkan Web Application Firewall (WAF)
// AMAN — pakai prepared statement
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
$stmt->execute([$username]);

5 Tools Wajib untuk Pemula di Cybersecurity
Bingung mulai dari mana? Berikut 5 tools yang wajib kamu kenal dan kuasai sebagai pemula di dunia cybersecurity — dari Nmap untuk reconnaissance sampai Burp Suite untuk web testing...
toolsbeginnernmap
Baca

Kenapa Perlu Tahu Tools Dulu?

Belajar cybersecurity tanpa mengenal tools ibarat memasak tanpa peralatan dapur. Tools ini bukan untuk "hack" seenaknya — tapi untuk memahami sistem, mencari kelemahan, dan belajar bertahan.

1. Nmap — Network Scanner

Nmap adalah tool paling fundamental untuk reconnaissance. Digunakan untuk men-scan jaringan, menemukan host aktif, dan mengidentifikasi service yang berjalan.

# Scan basic
nmap 192.168.1.1

# Scan dengan deteksi OS dan service version
nmap -sV -O target.com

# Scan semua port
nmap -p- target.com

2. Burp Suite — Web Proxy

Tool wajib untuk web application testing. Bisa intercept, modify, dan replay HTTP request. Versi Community gratis sudah cukup untuk belajar.

3. Wireshark — Packet Analyzer

Untuk menganalisis traffic jaringan secara real-time. Sangat berguna untuk forensics challenge di CTF dan memahami protokol jaringan.

4. Metasploit Framework

Framework exploitation yang powerful. Gunakan hanya di lab sendiri atau dengan izin! Berisi ribuan exploit yang sudah terdokumentasi.

5. John the Ripper — Password Cracker

Tool untuk crack password hash. Sering dibutuhkan di challenge CTF kategori crypto dan forensics.

# Crack hash dengan wordlist rockyou
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
💡 Semua tools di atas tersedia di Kali Linux — distro Linux yang dirancang khusus untuk penetration testing.

Pengalaman Pertama Ikut picoCTF — Seru tapi Bikin Pusing!
picoCTF adalah kompetisi CTF yang diselenggarakan Carnegie Mellon University, cocok banget untuk pemula seperti saya. Ini cerita pengalaman pertama saya ikut CTF, apa yang berhasil, apa yang gagal, dan pelajaran yang saya dapat...
ctfpicoctfpengalaman
Baca

Awal Mula Kenal CTF

Saya pertama kali dengar tentang CTF (Capture The Flag) dari teman sekelas yang sering ngomongin soal hacking. Katanya ini cara seru buat belajar cybersecurity sambil kompetisi. Langsung deh saya coba daftar picoCTF 2024.

Persiapan Sebelum Kompetisi

Jujur, waktu itu persiapan saya minim banget. Yang saya lakukan cuma:

  • Install Kali Linux di VirtualBox
  • Baca-baca dasar tentang kategori challenge: web, crypto, forensics
  • Latihan di platform TryHackMe beberapa minggu sebelumnya

Challenge yang Berhasil Saya Selesaikan

Dari total challenge yang ada, saya berhasil solve sekitar 40%. Yang paling berkesan adalah:

  • Obedient Cat — super mudah, tinggal cat file flagnya
  • Python Wrangling — baru tau ada flag -d untuk decode
  • Caesar — bikin script bruteforce sederhana

Yang Gagal dan Pelajarannya

Banyak challenge yang saya stuck, terutama kategori binary exploitation (pwn). Ini memang butuh pemahaman assembly dan memory management yang belum saya punya.

"Gagal di CTF bukan berarti kamu bodoh — itu artinya ada hal baru yang bisa dipelajari."
🎯 Tips: Mulai dari challenge yang poinnya kecil dulu. Biasanya itu yang paling mudah dan bisa boost semangat untuk terus solve yang lebih susah.
